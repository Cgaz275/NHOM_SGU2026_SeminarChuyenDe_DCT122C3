# Các Tài Liệu Source of Truth

Dưới đây là danh sách tất cả các tài liệu chính thức (Source of Truth) của dự án Persona-Based Website for Digital Twin Card. Mỗi tài liệu có vai trò riêng, giúp nhóm và các AI Agent làm việc nhất quán và hiệu quả.

| **Tài liệu**                              | **Tên đầy đủ**                          | **Ý nghĩa**                                                                                                                 | **Khi nào sử dụng**                                                                                                          |
| ----------------------------------------- | --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **PRD.pdf**                               | Product Requirement Document            | Tài liệu định nghĩa yêu cầu sản phẩm, mục tiêu, user stories, use cases, phạm vi (In/Out Scope) và luồng nghiệp vụ.         | \- Khi cần hiểu rõ yêu cầu chức năng - Làm cơ sở để thiết kế tính năng mới - Kiểm tra xem tính năng đã làm đúng yêu cầu chưa |
| **GUI DESIGN.pdf**                        | GUI Design Document                     | Tài liệu thiết kế giao diện chi tiết (Design System, màu sắc, typography, bố cục các màn hình, component style, UX rules).  | \- Thiết kế hoặc code Frontend - Review UI/UX - Đảm bảo giao diện nhất quán với thiết kế                                     |
| **Architecht & Database.pdf**             | Architecture & Database Design Document | Tài liệu mô tả kiến trúc hệ thống, công nghệ stack, cấu trúc Database (Firestore), Data Models, API Design, Security Rules. | \- Thiết kế hoặc code Backend - Làm việc với Database - Xây dựng API - Review kiến trúc hệ thống                             |
| **TEST PLAN.pdf**                         | Test Plan Document                      | Tài liệu lập kế hoạch kiểm thử, phạm vi test, chiến lược test, test environment, deliverables.                              | \- Lập kế hoạch và thực hiện kiểm thử - Viết Test Cases - Báo cáo kết quả test                                               |
| **Communication and Prompt Strategy.pdf** | Communication and Prompt Strategy       | Tài liệu hướng dẫn cách giao tiếp, prompt engineering, phân vai trò Agent, quy trình làm việc với AI và Prompt Templates.   | \- Làm việc với AI Agents (BuilderIO, Gemini, GPT...) - Viết Prompt hiệu quả - Đảm bảo chất lượng output từ AI               |


### Tóm tắt

- **PRD.pdf** — Xác định **yêu cầu sản phẩm** và phạm vi dự án.
- **GUI DESIGN.pdf** — Tiêu chuẩn **thiết kế giao diện** và trải nghiệm người dùng.
- **Architecht & Database.pdf** — Kiến trúc hệ thống, cơ sở dữ liệu và thiết kế Backend.
- **TEST PLAN.pdf** — Chiến lược và kế hoạch **kiểm thử** toàn hệ thống.
- **Communication and Prompt Strategy.pdf** — Hướng dẫn **giao tiếp và prompt** với AI Agents.

**Quy tắc sử dụng:**  
Khi thực hiện bất kỳ task nào, các AI Agent và thành viên nhóm **phải tham chiếu** các tài liệu tương ứng để đảm bảo tính nhất quán.
